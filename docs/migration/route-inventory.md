## Uncertainties

- `gatsby-node.js` contains no `createPages`, `createPage`, or redirect code, and `gatsby-config.js` contains no page rewrites. No additional programmatic page routes or redirects were found in those files.
- The inventory treats each `src/pages/*.tsx` file as a trailing-slash Gatsby route. Runtime hosting normalization could expose equivalent non-trailing-slash URLs, but that is not specified in the repository code inspected.
- Reusable components have conditional variants. Where a component imports several forms, the table names the form whose rendered variant can be determined from the page’s props; a component’s unused alternative variants are called out only when relevant.
- `MasterclassAgenda` contains both `meta-ads-zainteresowani.html` and `ogarnij-swoje-adsy-zainteresowani.html`; the exact form is determined by its `version` branch. The page rows identify the branches that can be established from the source, but do not infer forms from versions that are not rendered.
- Prices and checkout URLs embedded in imported `values/*` modules, reusable components, or raw HTML files are not exhaustively repeated unless they are directly relevant and code-visible in the inspected page/component path. Global GDPR Google Analytics, Google Tag Manager, and Facebook Pixel configuration is site-wide rather than page-specific, so it is not repeated in every row.
- The API rows are included because `src/api/` is an explicit inventory source, although they are Gatsby Functions rather than document/page routes.
